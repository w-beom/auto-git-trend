# Collapse Card Details Design

## Goal

전체 목록 카드에서 긴 상세 요약을 기본적으로 접은 상태로 보여주고, 카드 우측 상단의 화살표 버튼으로만 전체 문단을 펼쳐 볼 수 있게 한다.

## Scope

- 적용 대상은 `RepositoryCard`의 `list` variant만이다.
- `feature` variant는 현재처럼 모든 문단을 그대로 노출한다.
- 각 카드의 펼침 상태는 카드 내부 로컬 상태로만 관리한다.
- 페이지 이동, 새로고침, 서버 렌더 재실행 후에는 다시 기본 접힘 상태로 돌아간다.

## Interaction Design

### Default state

- `list` 카드의 summary 영역은 첫 문단만 렌더링한다.
- summary가 두 문단 이상일 때만 토글 버튼을 노출한다.
- summary가 한 문단뿐이면 토글 버튼을 숨기고 현재처럼 정적 카드로 렌더링한다.

### Expanded state

- 우측 상단 화살표 버튼을 누르면 summary의 모든 문단을 렌더링한다.
- 같은 버튼을 다시 누르면 첫 문단만 보이는 기본 상태로 돌아간다.
- 각 카드는 서로 영향을 주지 않는다. 한 카드를 펼쳐도 다른 카드는 그대로 유지된다.

### Toggle affordance

- 토글 버튼은 카드 우측 상단에 배치한다.
- 카드 헤더는 왼쪽의 `rank / owner` 묶음과 오른쪽의 토글 버튼으로 재구성한다.
- 버튼은 텍스트 대신 화살표로 상태를 표현한다.
- 접근성 이름은 상태에 따라 `상세 펼치기`, `상세 접기`를 사용한다.
- 시각 상태도 접근성 상태와 맞춰 화살표 방향이 바뀌어야 한다.

## Component Design

### RepositoryCard

- `variant === "list"`일 때만 접힘 동작을 가진다.
- summary 문단 배열을 기존처럼 파싱한 뒤, 현재 상태에 따라 보여줄 문단 범위를 계산한다.
- 토글 가능한 카드라면 `useState`로 `isExpanded`를 관리한다.
- 요약 영역의 DOM 구조는 유지하되, 접힘 상태에서는 첫 문단만 포함한다.

### Layout changes

- `.repo-card__masthead`는 좌우 정렬이 가능한 구조를 유지하되, 토글 버튼이 우측 끝에 놓이도록 정렬 규칙을 추가한다.
- 토글 버튼은 카드 상단에서 별도 액션처럼 보여야 하지만, 기존 카드 디자인 톤은 유지한다.
- 긴 문단을 CSS로 잘라내는 방식 대신 실제 렌더링 문단 수를 제어한다. 이렇게 해야 “첫 문단만 표시” 요구를 정확히 만족할 수 있다.

## Error Handling And Edge Cases

- summary 문단이 비어 있거나 하나뿐인 경우에는 토글 버튼을 렌더링하지 않는다.
- `feature` 카드에는 토글 버튼을 렌더링하지 않는다.
- 접힘/펼침 상태는 URL, 저장소, 전역 상태와 연결하지 않는다.

## Testing

### Component tests

- 기본 렌더에서 `list` 카드가 첫 문단만 노출하는지 검증한다.
- 토글 버튼 클릭 시 모든 문단이 노출되는지 검증한다.
- 다시 클릭하면 첫 문단만 보이는 상태로 돌아가는지 검증한다.
- 문단이 하나뿐인 카드에서는 토글 버튼이 없는지 검증한다.
- `feature` 카드에서는 모든 문단이 계속 보이고 토글 버튼이 없는지 검증한다.

### Style tests

- 카드 헤더에 토글 버튼 배치를 위한 셀렉터가 존재하는지 검증한다.
- 토글 버튼의 collapsed/expanded 시각 상태를 위한 셀렉터가 존재하는지 검증한다.

## Out Of Scope

- 여러 카드를 한 번에 접거나 펼치는 전역 제어
- 펼침 상태 유지
- 애니메이션 기반 높이 전환
